Menu Fichier

Charger une partie ou plusieurs parties

Cliquer sur le menu Fichier/Charger une partie. Sélectionner les parties à charger. Pensez à regarder les options en bas de la fenêtre affichée, à savoir le chemin pour l'enregistrement de la base de données, le nom de fichier de la base de données et si c'est une session de test ou non.

 

Menu Expérience

Afficher l'écran d'accueil

Affiche l'écran d'accueil sur les postes clients (aussi appelés remotes). L'image affichée, ainsi que les textes peuvent être changés dans le fichier configparam du package configuration.

Charger un questionnaire de compréhension

Ouvre un boîte de dialogue permettant de sélectionner le fichier à charger. Un fichier de questionnaire de compréhension est un fichier xml avec une architecture bien définie. Pour créer un questionnaire de compréhension, il faut utiliser l'outil dédié, que l'on lance à partir du menu Outils/Créer ou éditer un questionnaire de compréhension. Lorsque le questionnaire est chargé une boîte de dialogue affiche le contenu du fichier. Pour lancer le questionnaire sur les postes clients il suffit de cliquer sur le menu Démarrer le questionnaire de compréhension.

Afficher le questionnaire final sur les postes

Lance le questionnaire final. Une capture d'écran de ce questionnaire se trouve dans le dossier screenshots.

Gains

Affiche une boîte de dialogue avec les gains des sujets, par poste. Il faut au préalable avoir ajouté les gains d'une ou plusieurs partie(s) à ces gains. Ces gains peuvent ensuite être imprimés, enregistrés ou affichés sur les postes clients. Dans ce dernier cas cela affiche sur les postes l'écran final, lequel non seulement informe le sujet de son gain pour l'expérience mais aussi lui offre un espace pour écrire des commentaires sur l'expérience.

 

Menu Parties

Lorsqu'une partie est chargé un sous-menu dédié à la partie est crée à cet endroit. Ce sous-menu, qui porte le nom de la partie ou un nom très proche, permet de piloter la partie, à savoir, en général: configurer, afficher les paramères, démarrer la partie et afficher les gains de la partie.

 

Menu Edition

Permet de vider la liste serveur et la liste client de l'onglet Expérience.

 

Menu Options

Arrêter après cette période

Arrête la partie après la période en cours. Passera donc directement au calcul des gains pour la partie.

Enregistrer le genre des clients/sujets dans l'application

Permet de définir pour chaque poste s'il s'agit d'un homme ou d'une femme. Le genre de chacun est fixé dans l'attribut genre de l'objet joueur. Cet attribut n'est pas enregistré dans la base de données, c'est simplement pour pouvoir l'utiliser dans le programme, par exemple pour former des groupes mixtes. Pour avoir le genre d'un client/sujet dans la base de données il faut passer par le questionnaire final.

Tirer au sort une partie parmi celles qui ont été lancées

Ouvre un boîte de dialogue avec la liste des parties qui ont été jouées effectivement par les clients. Sélectionner alors les parties parmi lesquelles faire un tirage au sort. Affiche alors dans la liste serveur (onglet expérience) le nom de la partie tirée au sort. Ne fait rien d'autre.

Afficher les gains de chaque partie

Ouvre un boîte de dialogue avec la liste des parties qui ont été jouées effectivement par les clients. Sélectionner alors les parties pour lesquelles on souhaite afficher les gains sur les postes clients. Une fois cet écran validé cela affiche sur les postes clients une boîte de dialogue d'information avec pour chaque partie sélectionnée le gain associé.

 

Menu Outils

Créer une nouvelle partie

Ouvre une boîte de dialogue qui permet de donner quelques informations sur la partie à créer (nom, répétitions, groupes ...). A la validation une boîte de dialogue s'ouvre pour demander où enregistrer les fichiers qui vont être créés. Normalement, mettre les fichiers dans le dossier le2m/parts afin que la partie en question apparaisse dans la liste des parties qu'il est possible de charger (menu Fichier/Charger une partie). Le logiciel va alors crée tous les fichiers nécessaire pour la partie et dans ces fichiers va écrire le code squelette d'une partie. Ce seront ces fichiers qu'il faudra éditer et compléter pour préciser le fonctionnement de la partie (étapes à lancer, calculs ...). Normalement la partie peut être lancée en l'état, elle fonctionne.

Créer ou éditer un questionnaire de compréhension

Comme son nom l'indique, permet de créer et/ou d'éditer un questionnaire de compréhension qui sera alors utilisable pour l'expérience, via le menu Expérience/Charger un questionnaire de compréhension. Un questionnaire de compréhension est un fichier xml avec une structure particulière. Cet éditeur permet, de manière simple et graphique, de créer ce fichier. Il est aussi possible de charger un fichier existant pour soit le modifier soit le tester. Dans ce dernier cas le questionnaire est lancé localement et ce que l'on voit correspond exactement à ce que verront les sujets.

Extraire des données

Ouvre un boîte de dialogue qui permet de sélectionner un fichier sqlite créer avec LE2M. Une fois le fichier sélectionné, ouvre une boîte de dialogue qui affiche les parties contenus de la base de données. Il suffit alors de sélectionner les parties à exporter. A la validation ouvre une boîte de dialogue pour choisir le dossier dans lequel enregistrer les fichiers qui vont être créés. Le programme créé alors trois fichiers

  1. le fichier sessions.csv: contient la liste des sessions qui ont été exportées, cad toutes celles qui ne sont pas des tests.
  2. le fichier joueurs.csv: contient la liste de tous les joueurs qui ont participé et pour lesquels les données ont été extraites
  3. le fichier data.csv: contient les données pour toutes les sessions et tous les joueurs listés dans les deux fichiers précédents, pour toutes les parties sélectionnées précédemment. Dans ce fichier il y a une ligne par joueur et par répétition. Sur cette ligne il y a toutes les variables de toutes les parties.

Lancé de dé

Ouvre une boîte de dialogue qui simule, en 2D, un lancé de dé. Démarrage et arrêt du défilement se fait manuellement. L'order d'affichage des faces du dé est aléatoire. Le résultat du lancé est affiché sur la liste serveur.

Nombre aléatoire

Ouvre une boîte de dialogue qui permet de fixer la valeur minimum et la valeur maximum. Démarrage et arrêt du défilement se fait manuellement. L'ordre d'affichage des nombres est aléatoire. Le résultat est affiché sur la liste serveur.

Pile ou face

Ouvre une boîte de dialogue qui simule, en 2D, un lancé de pièce de 1 euro. Le démarrage et l'arrêt du défilement se fait manuellement. Le résultat du lancé est affiché sur la liste serveur.
